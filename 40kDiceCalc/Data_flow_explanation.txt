# Data Flow Explanation

This document outlines the high-level Extract, Transform, Load (ETL) pipeline used to generate and visualize probability data.

## 1. High-Level Pipeline Architecture

[User Input (UI) or API Search]
      |
      v
[Event Manager or Fetch Router]       * Intercepts inputs, fetches API data, and broadcasts global events
      |
      v
[Extract State]                       * Parses DOM for Weapon/Unit profiles or translates API payloads
      |
      v
[Web Worker]                          * Background Thread: Runs 100,000 waterfall iterations per profile
      |
      v
[Logic Engine]                        * Executes pure JS math sequentially (Hits > Wounds > Saves > Damage); Returns structured JSON distributions
      |
      v
[SQL Database]                        * In-Memory SQLite (sql.js). INSERTS raw distribution rows
      |
      v
[SQL Query]                           * SELECT rows grouped by Category (Hit/Wound/Save) and Modifier
      |
      v
[Data Processing]                     * Sorts array averages for Leaderboard rendering
      |
      v
[Chart Rendering]                     * Chart.js maps SQL rows to X/Y coordinates
      |
      v
[DOM Output]                          * Rendered UI Report Cards and Leaderboards

## 2. Web Worker Execution Flow

[Main Thread (app.js)]
      |
      * postMessage({ weapons, target, iterations })
      v
[webWorker.js]
      |
      * Imports runSimulation()
      * Executes waterfall loop (resolving hits, passing successes directly to wounds)
      |
      * postMessage(results)
      v
[Main Thread (app.js)]
      |
      v
(Data goes to SQL Engine)

## 3. Database Aggregation Flow

Application Logic
|
* Extract Output
  * JSON object: { "4": 1200, "5": 3000 }
|
* Prepare Statement (db-manager.js)
  * "INSERT INTO simulation_runs (modifier_name, category, value, occurrence_count) VALUES (?, ?, ?, ?)"
|
* Execute Query
  * stmt.run(["Lethal", "Hit", 4, 1200])
|
* SQLite Engine (WASM)
  * Update Storage (RAM)
